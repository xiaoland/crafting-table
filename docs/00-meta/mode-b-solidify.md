# Mode B - Solidify

## Role

Use when findings must be restated into stable claims, decisions, owners, or promotion candidates.

## Guardrails

- Do not start coding while durable ownership is still ambiguous.
- Do not promote unstable guesses into durable docs.

## Read-do steps

1. Gather the current findings, evidence, and assumptions.
2. Decide what is stable enough to promote and what must stay in `tasks/`.
3. Restate target, scope, guardrails, and verification.
4. Confirm the durable owner for each stable truth.
5. Hand off to Execute or return to Explore if ownership is still unclear.

## Exit criteria

- Durable ownership is explicit.
- Verification is explicit.
- The next edit is safe to perform.
