# Cross-domain interactions

- interaction pair: `Goal Forest` <-> `work session`
  semantic contract: Goal Forest gives sessions placement and nearby context; a session remains the execution object.
  shared language: node, linked session, active session, related work

- interaction pair: `capture` <-> `work session`
  semantic contract: a capture may attach to a session immediately or later without losing append-first speed.
  shared language: attach, save now, classify later, continuity

- interaction pair: `capture` <-> `Goal Forest`
  semantic contract: captures may link into Goal Forest when the right context becomes clear.
  shared language: node, link, placement

- interaction pair: `Remote Control` <-> `work session`
  semantic contract: remote work may begin from or attach to a session, and minimal continuity returns to the session.
  shared language: attach, linked session, outcome note, return path

- interaction pair: `Remote Control` <-> `host profile`
  semantic contract: Remote Control uses workspace-scoped host profiles instead of copying connection definitions into each session.
  shared language: host profile, saved host, connect, reuse

- interaction pair: `Codex Remote` <-> `Companion`
  semantic contract: Codex Remote speaks a small CraftingTable-owned host contract; Companion adapts that contract to Codex app-server, Desktop Scout, and host-specific process details.
  shared language: host, Companion, project, thread, turn, stream event, permission mode
